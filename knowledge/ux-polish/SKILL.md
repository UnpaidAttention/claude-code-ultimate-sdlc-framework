### Step 4: Error Message Enhancement
```markdown
## Error Message Polish

| Error | Current | Improved | Why Better |
|-------|---------|----------|------------|
| [Error type] | "[Current text]" | "[Improved text]" | [Reason] |

### Good Error Message Formula
1. What happened (clear, specific)
2. Why it happened (if helpful)
3. How to fix it (actionable)
4. Encouraging tone (not blaming)

**Example**:
- Before: "Error: Invalid input"
- After: "The email address needs an @ symbol. Try entering it like name@example.com"
```

### Step 5: Loading State Polish
```markdown
## Loading States

| Scenario | Duration | Current | Improved |
|----------|----------|---------|----------|
| Page load | >300ms | [Current] | [Skeleton/spinner] |
| Form submit | >500ms | [Current] | [Button state + spinner] |
| Data fetch | >1000ms | [Current] | [Progress indicator] |

### Loading State Guidelines
- <300ms: No indicator needed
- 300ms-1s: Subtle spinner
- 1-10s: Progress indicator
- >10s: Progress with status updates
```

### Step 6: Empty State Polish
```markdown
## Empty States

| Location | Current | Improved |
|----------|---------|----------|
| [List view] | [Current/blank] | [Helpful message + action] |
| [Search results] | [Current] | [Suggestions + alternatives] |
| [Dashboard] | [Current] | [Getting started guide] |

### Good Empty State Formula
1. Acknowledge the empty state
2. Explain why it's empty (if not obvious)
3. Guide to next action
4. Keep it friendly
```
