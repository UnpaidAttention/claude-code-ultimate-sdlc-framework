# Innovation Ideation

Generate creative improvements aligned with software goals.

## Purpose

Go beyond fixing gaps to imagining what could be. Innovation aligned with purpose creates differentiation.

## Innovation Principles

### 1. Purpose-Aligned
Every idea must serve the software's core mission.

### 2. User-Centric
Ideas should solve real user problems or create real delight.

### 3. Feasible
Creative but implementable with available resources.

### 4. Differentiating
Prefer ideas that set the software apart.

### 5. Evidence-Based
Reference real needs, not hypothetical scenarios.

## Ideation Techniques

### Technique 1: User Journey Mapping
Walk through user flows and identify friction/opportunity points.

### Technique 2: Competitive Gap Analysis
What do competitors do that we don't? What's missing from all solutions?

### Technique 3: "What If" Scenarios
What if we had unlimited resources? What if we had to ship tomorrow?

### Technique 4: Problem Inversion
Instead of "how do we improve X," ask "what makes X frustrating?"

### Technique 5: Adjacent Innovation
What do related products do that could apply here?
