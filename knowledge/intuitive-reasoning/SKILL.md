# Intuitive Reasoning

Harnesses pattern recognition and experiential intuition for software assessment. Trusts "gut feelings" while validating them systematically.

## Understanding Intuition

### What Intuition Is
- Pattern recognition from accumulated experience
- Rapid processing of multiple subtle signals
- Holistic assessment before detailed analysis
- Detection of anomalies from expected patterns

### What Intuition Is Not
- Random guessing
- Unfounded opinion
- Excuse to skip analysis
- Always correct

### When Intuition Excels
- Initial exploration and orientation
- Identifying investigation areas
- Detecting "something's wrong"
- Holistic quality assessment

### When Analysis Must Lead
- Verifying intuitive signals
- Documenting findings
- Making formal recommendations
- Quantifying issues

## The Intuition Protocol

### 1. Acknowledge the Signal
When something feels off:
```markdown
## Intuitive Signal

**Trigger**: [What prompted the feeling]
**Initial Reaction**: [Describe the gut response]
**Intensity**: [Strong / Moderate / Subtle]
```

### 2. Pause and Reflect
Before investigating, capture:
- What specifically feels wrong?
- Where is the feeling located (which feature/screen)?
- Is it familiar from past experience?

### 3. Pattern Match
Connect to prior experience:
- Have I seen this before?
- What situation does this remind me of?
- What usually causes this feeling?

### 4. Hypothesize
Form testable hypotheses:
- What could be causing this feeling?
- What specific issue might exist?

### 5. Verify
Test hypotheses systematically:
- Gather evidence
- Run specific tests
- Confirm or refute

### 6. Learn
Document for future:
- Was intuition correct?
- What signals were accurate?
- What to remember?

## Pattern Recognition

### Positive Patterns (Things Feel Right)
- Consistent interaction models
- Predictable system responses
- Familiar navigation structures
- Appropriate feedback
- Clear visual hierarchy
- Logical information flow

### Warning Patterns (Things Feel Wrong)
- Inconsistent behavior
- Unexpected states
- Confusing navigation
- Missing feedback
- Visual dissonance
- Broken expectations

### Anti-Pattern Recognition
Common issues intuition detects:
- Unnecessary complexity
- Missing error handling
- Poor performance
- Security red flags
- Usability friction
- Incomplete features

## Holistic Assessment

### First Impressions Capture
Within the first few minutes, note:
- Overall quality impression
- Immediate friction points
- Moments of delight
- Confusion instances
- Trust signals

| Dimension | Rating | Notes |
|-----------|--------|-------|

| Fast vs. Sluggish | 1-5 | |

### Flow Assessment
- Does the product flow naturally?
- Are there jarring transitions?
- Do features connect logically?
- Is there rhythm to the experience?

## Intuition Calibration

### Building Reliable Intuition
- Seek feedback on intuitive assessments
- Track accuracy over time
- Study false positives and negatives
- Learn from expert assessments

### Calibration Questions
After each intuitive assessment:
- Was my intuition accurate?
- What signals led me astray?
- What signals were reliable?
- How can I improve?

## Balancing Intuition and Analysis

### Integration Approach
```
Intuition → Surfaces Candidates → Analysis → Confirms/Refutes
```

1. Use intuition for exploration and detection
2. Use analysis for verification and documentation
3. Trust intuition to guide attention
4. Trust analysis for conclusions

### When to Override Intuition
- Evidence contradicts feeling
- Bias might be influencing
- Stakes are high
- Intuition is uncertain

### When to Trust Intuition
- Strong, clear signal
- Pattern is familiar
- Stakes allow experimentation
- Quick decision needed
